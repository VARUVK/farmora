ROLE

You are a senior software architect and product auditor.

Your task is to review and verify the existing codebase of an app called Farmora.

Do NOT generate new features unless required to fix a critical issue.
Your primary task is evaluation and validation, not rebuilding.

CONTEXT

Farmora is a web-first agricultural intelligence platform for India with:

Farmers

Traders

Private Admin (owner-only)

The app includes:

OTP authentication

Role-based access

AI advisory (SELL / HOLD / WAIT)

Simulation & profit estimation

Notifications (App, SMS, WhatsApp placeholder)

Multi-language support (English, Tamil, Hindi)

AUDIT OBJECTIVES (MANDATORY)
1️⃣ RUNNABILITY CHECK

Verify the app starts without errors

Identify missing environment variables

Report blocking runtime issues first

2️⃣ PROJECT STRUCTURE REVIEW

Check that:

Frontend and backend are separated

Backend logic is modular

No single-file monolith exists

Flag any architectural smells.

3️⃣ AUTH & ROLE SAFETY

Verify:

OTP-based authentication exists (even mocked)

User roles are stored and enforced server-side

Admin routes are hidden and protected

Flag any role-leak risks.

4️⃣ FARMER FEATURE VALIDATION

Confirm farmer can:

Select multiple crops

Select state & district (India)

View mandi prices

Receive advisory with:

Recommendation

Confidence %

Risk level

Clear reasons

5️⃣ TRADER FEATURE VALIDATION

Confirm trader can:

Select crops & regions

Discover farmers only with consent

Express interest or contact intent

Flag privacy violations.

6️⃣ AI ADVISORY SAFETY CHECK (CRITICAL)

Inspect advisory logic and ensure:

Deterministic rules exist before AI text

No guaranteed profit language

No exact price predictions

Uncertainty and risk are shown

❌ If unsafe language exists, flag as CRITICAL RISK.

7️⃣ SIMULATION & PROFIT LOGIC

Verify:

Simulation horizon (N days) exists

Expected price is estimated

Profit = price − input costs

Risk band shown

Flag fake or random logic.

8️⃣ DATA SOURCE & FALLBACK CHECK

Ensure:

Market price source is documented

Weather data referenced

Fallback logic exists when data missing

Confidence or freshness indicator shown

9️⃣ NOTIFICATION SYSTEM REVIEW

Verify:

Notification logic is abstracted

WhatsApp is placeholder-only

No provider is hardcoded deep into business logic

🔟 LANGUAGE & REGION SUPPORT

Confirm:

English default UI

Tamil & Hindi support exists or scaffolded

Regions are not hardcoded to a single state

OUTPUT FORMAT (STRICT)

Return your audit in this exact structure:

✅ PASSED

Bullet list of correctly implemented items

⚠️ NEEDS IMPROVEMENT

Bullet list with file references

❌ CRITICAL ISSUES

Bullet list (security, safety, or trust risks)

🧠 OVERALL SCORE

Score out of 10

One-line verdict:
“MVP-ready” / “Salvageable” / “Do not proceed”

🛠️ NEXT ACTIONS

Top 3 fixes in priority order

RULES

Do not rewrite the whole app

Do not add new features

Focus on correctness, safety, and MVP readiness

Be brutally honest
